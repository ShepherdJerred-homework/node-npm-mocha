1. Create a Node project.
    • You should use “npm init” to create your package.json file.
    • You should provide values for all entries except “git repository” and “keywords”.
    • Your entry point should be main.js.
    • You should use mocha as your test command.
  2. Create a main.js file. Your file should export the following three functions:
    1. A function named repeat that takes two parameters: a string and a number. It should create a string that consists of the given string repeated the given number of times. If the number is less than 1 then just return the empty string.
      ?
      repeat("howdy", 3) == "howdyhowdyhowdy"
      Remember that the + operator can concatenate two strings.
      ?
      "hi" + "there" == "hithere"
    2. A function named runningSum that takes one parameter: an array of numbers. It should return a new array of numbers such that the ith element is the sum of the first i numbers of the original array. Do not modify the original array.
      ?
      sum([ 2, 4, 5, 9 ]) == [ 2, 6, 11, 20 ]
    3. A function named slice that takes two parameters: an object and an array of strings. It should return a new object that is a copy of the given object except that it only contains the properties contained in the given array of strings. Do not modify the original array.
      ?
      let o = { a: 3, b: 8, c: 14, d: 6, e: 21, f: 15, g: 2, h: 5 };
       
      slice(o, ["a", "f", "c", "x", "e"]) == { a: 3, f: 15, c: 14, e: 21 }
      You might find it helpful to use the array's built-in indexOf function. It returns the first index of the given element if it is in the array or -1 if it is not in the array.
      ?
      [ 10, 20, 30, 40, 30 ].indexOf(30) == 2;
       
      [ 10, 20, 30, 40, 30 ].indexOf(50) == -1;
  3. Create a test.js file. Your file should contain tests for the functions you wrote
    • Create a describe group for each function
    • You should have at least three tests for each funciton
    • Remember that your test cases should describe the behavior of your functions: what does it do?
    • Remember to test special cases: what if the array is empty? what if the number is negative?, what if none of the keys are in the object? etc.
  4. Once you have all your test cases working you should delete your node_modules directory, zip up your package folder, and submit it.

    I should be able to unzip your project folder, run “npm install”, then run “npm test” to verify that your program passes all its test cases.
